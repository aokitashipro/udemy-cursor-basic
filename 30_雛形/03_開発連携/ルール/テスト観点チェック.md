# テスト観点チェック

## 必須確認項目
- 機能要件/非機能要件/境界値
- 異常系/セキュリティ/パフォーマンス

## 詳細チェックリスト

### 機能テスト
- [ ] 正常系の基本機能が動作する
- [ ] 入力値の境界値（最小・最大・空）をテスト
- [ ] 異常値入力時の適切なエラーメッセージ表示
- [ ] 権限に応じた機能制限の確認

### 非機能テスト
- [ ] レスポンス時間が要件を満たす（3秒以内など）
- [ ] 同時接続数の上限確認
- [ ] データベース負荷時の動作確認
- [ ] セキュリティ要件（認証・暗号化）の確認

### ユーザビリティ
- [ ] 操作手順が直感的で分かりやすい
- [ ] エラー時の回復方法が明確
- [ ] モバイル・PC両対応の表示確認

## 品質基準
**良い例**：「ユーザー登録で同一メールアドレス入力時に『既に登録済みです』エラー表示」
**悪い例**：「エラーハンドリングを確認」
